# BodyHealthPro Development Guidelines

## 🎯 Core Principles (ALWAYS APPLY)

### 1. Responsive-First (MANDATORY)
**Every implementation MUST include responsive design from the start.**

- Mobile-first approach: start with base Tailwind classes
- Add `sm:`, `md:`, `lg:`, `xl:` breakpoints progressively
- Minimum touch target: 44px on mobile
- No horizontal scrolling on any device
- Test on 320px, 768px, 1920px viewports

**Reference**: Use skill `bodyhealthpro-responsive-dev` for guidance

### 2. Design System Compliance (MANDATORY)
**All UI must use BodyHealthPro design system components.**

#### Color Palette
- **Accent**: `#a9ff2e` (bright green)
- **Primary Text**: `#d8ffe8` (light green)
- **Secondary Text**: `text-gray-400`
- **Background**: `rgba(9,16,12,0.86)` (dark)
- **Borders**: `rgba(169,255,46,0.2)` (green border)

#### Components to Use
- `Button` from `@/components/ui/button`
- `Card`, `CardContent`, `CardHeader` from `@/components/ui/card`
- `CheckboxComponent` from `@/components/ui/checkbox`
- `Input` from `@/components/ui/input`
- `Collapsible` from `@/components/ui/collapsible`

**Reference**: Use skill `bodyhealthpro-design-patterns` for component patterns

### 3. Component Quality Standards
Every component must:

- ✅ Be fully typed with TypeScript (no `any`)
- ✅ Include responsive classes from day one
- ✅ Follow mobile-first grid pattern: `grid grid-cols-1 sm:grid-cols-2 md:grid-cols-3`
- ✅ Have documented props with JSDoc
- ✅ Use semantic HTML where possible
- ✅ Include accessibility attributes (`title`, `aria-*`)
- ✅ Pass ESLint without warnings: `npm run lint`
- ✅ Compile without errors: `npm run build`

### 4. State Management
- Use React hooks for local state (`useState`)
- Use React Query for server state (`useQuery`, `useMutation`)
- Use custom hooks for shared logic (e.g., `useAvaliacoes`, `useAuth`)
- Implement proper loading/error states

### 5. Database Integration
- Always include `tenant_id` for RLS compliance
- Add `criado_por` audit field for tracking
- Validate data before INSERT/UPDATE
- Use typed service functions (e.g., `createAvaliacao()`)

## 📱 Responsive Template (Copy-Paste Ready)

```typescript
import { cn } from '@/lib/utils'
import { Card, CardContent, CardHeader } from '@/components/ui/card'
import { Button } from '@/components/ui/button'

interface ComponentProps {
  title: string
  items: Item[]
  onAction?: () => void
}

/**
 * Responsive component template
 * Responsive breakpoints:
 * - Mobile (320px+): Single column, stacked buttons
 * - Tablet (768px+): Two columns, side-by-side buttons
 * - Desktop (1024px+): Three columns, full layout
 */
export function ComponentName({ title, items, onAction }: ComponentProps) {
  return (
    <div className="w-full px-3 sm:px-4 md:px-6">
      {/* Responsive header */}
      <h1 className="text-lg sm:text-xl md:text-2xl font-bold text-[#d8ffe8] mb-3 sm:mb-4 md:mb-6">
        {title}
      </h1>

      {/* Responsive grid */}
      <div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-3 sm:gap-4 md:gap-6 mb-4 sm:mb-6">
        {items.map((item) => (
          <Card key={item.id}>
            <CardContent className="p-3 sm:p-4">
              <p className="text-sm sm:text-base text-[#d8ffe8] font-semibold">{item.name}</p>
              <p className="text-xs sm:text-sm text-gray-400 mt-1 sm:mt-2">{item.description}</p>
            </CardContent>
          </Card>
        ))}
      </div>

      {/* Responsive buttons */}
      <div className="flex flex-col sm:flex-row gap-2 sm:gap-3 w-full">
        <Button className="w-full sm:flex-1" onClick={onAction}>
          Primary Action
        </Button>
        <Button variant="outline" className="w-full sm:flex-1">
          Secondary Action
        </Button>
      </div>
    </div>
  )
}
```

## ❌ Anti-Patterns (NEVER DO)

```typescript
// ❌ NEVER: Fixed widths
<div className="w-[800px]">

// ❌ NEVER: Only desktop-sized grids
<div className="grid grid-cols-4 gap-6">

// ❌ NEVER: No spacing responsiveness
<div className="p-6 gap-4">

// ❌ NEVER: Buttons without width classes on mobile
<button className="flex-1">
  {/* This breaks layout on narrow screens */}
</button>

// ❌ NEVER: Using light text on light backgrounds
<div className="bg-white text-gray-200">

// ❌ NEVER: Typography without breakpoints
<h1 className="text-4xl">
  {/* Too big on mobile */}
</h1>

// ❌ NEVER: Untyped components
function MyComponent(props: any) {
  // No type safety
}
```

## ✅ Best Practices (DO THIS)

```typescript
// ✅ YES: Mobile-first responsive grid
<div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-3 sm:gap-4 lg:gap-6">

// ✅ YES: Responsive padding
<div className="p-3 sm:p-4 md:p-6">

// ✅ YES: Responsive typography
<h1 className="text-lg sm:text-xl md:text-2xl">

// ✅ YES: Responsive buttons
<Button className="w-full sm:flex-1">Action</Button>

// ✅ YES: Fully typed props
interface ComponentProps {
  title: string
  items: Item[]
  onAction: () => void
}

// ✅ YES: Dark theme colors
<div className="bg-[rgba(9,16,12,0.86)] text-[#d8ffe8]">
```
