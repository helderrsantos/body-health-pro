# BodyHealthPro Implementation Agent

## 🎯 Mission
Guarantee that **EVERY** implementation in BodyHealthPro includes responsive design, design system compliance, and quality standards from the ground up.

## ✅ Automatic Checks (Applied to Every Implementation)

### Check 1: Responsive-First Design
```
QUESTION: "Does this implementation include mobile-first responsive design?"
- Includes `grid grid-cols-1 sm:grid-cols-2` pattern? ✓
- Uses responsive padding/margins? ✓
- Has no fixed widths? ✓
- Touch targets 44px+ on mobile? ✓
```

### Check 2: Design System Compliance
```
QUESTION: "Does this use BodyHealthPro design system?"
- Using color palette (#a9ff2e, #d8ffe8, rgba(9,16,12,0.86))? ✓
- Using ui/ components (Button, Card, Checkbox, Input)? ✓
- Dark theme applied? ✓
- Lucide icons used? ✓
```

### Check 3: Code Quality
```
QUESTION: "Does this meet code quality standards?"
- TypeScript fully typed (no `any`)? ✓
- ESLint passes? ✓
- Build succeeds? ✓
- No console errors? ✓
```

## 🚀 Pre-Implementation Checklist

Before starting ANY feature, this agent will:

1. **Verify Responsive Plan**
   - [ ] Mobile layout (320px) described
   - [ ] Tablet layout (768px) described
   - [ ] Desktop layout (1920px) described
   - [ ] Touch targets identified
   - [ ] Typography scaling defined

2. **Check Design System**
   - [ ] Color palette selected
   - [ ] Components identified (Button, Card, etc.)
   - [ ] Icons from lucide-react chosen
   - [ ] Dark theme confirmed

3. **Load Required Skills**
   - [ ] `bodyhealthpro-responsive-dev` loaded
   - [ ] `bodyhealthpro-design-patterns` loaded
   - [ ] Project `.instructions.md` reviewed

## 🔄 During Implementation

### Automatic Validation
This agent will:

1. **Generate responsive template** using the provided copy-paste snippet
2. **Apply design system colors** automatically
3. **Add breakpoints** for all layout changes
4. **Ensure touch targets** are 44px+ on mobile
5. **Include accessibility** attributes
6. **Type all props** in TypeScript

### Code Review Patterns
```typescript
// ❌ RED FLAG: Fixed grid
grid-cols-2
// ✅ SUGGESTION: Use responsive grid
grid-cols-1 sm:grid-cols-2

// ❌ RED FLAG: No padding responsiveness
p-6
// ✅ SUGGESTION: Add breakpoints
p-3 sm:p-4 md:p-6

// ❌ RED FLAG: Fixed width button
flex-1
// ✅ SUGGESTION: Mobile-first
w-full sm:flex-1
```

## 🚫 Blocking Issues

This agent will **BLOCK** submission if:

1. ❌ No responsive breakpoints
2. ❌ Fixed widths used (e.g., `w-[800px]`)
3. ❌ Light colors on light backgrounds
4. ❌ Touch targets < 44px on mobile
5. ❌ ESLint errors
6. ❌ Build failures
7. ❌ TypeScript `any` types
8. ❌ Not using design system components
9. ❌ Horizontal scrolling on mobile
10. ❌ No accessibility attributes

## ✅ Approval Criteria

This agent will **APPROVE** when:

1. ✅ Responsive breakpoints applied (mobile-first)
2. ✅ Design system colors used
3. ✅ Design system components used
4. ✅ Dark theme applied
5. ✅ Touch targets 44px+ on mobile
6. ✅ No horizontal scrolling
7. ✅ ESLint passes (0 errors)
8. ✅ Build passes (0 errors)
9. ✅ TypeScript fully typed
10. ✅ Tested on 320px, 768px, 1920px
