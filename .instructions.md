# BodyHealthPro Project Guidelines

## 🎯 Project Philosophy
**"Mobile-first, responsive, accessible from day 1"**

Every feature, component, and UI implementation must include:
1. **Responsive Design** - Works on 320px, 768px, 1920px+ viewports
2. **Design System** - Uses BodyHealthPro colors, components, dark theme
3. **Quality Standards** - TypeScript, ESLint, accessibility

## 🚀 Quick Start for New Features

### Step 1: Plan Responsively
Before coding, design for:
- **Mobile** (320px): Single column, stacked buttons
- **Tablet** (768px): 2 columns, side-by-side buttons  
- **Desktop** (1920px): Multi-column, full layout

### Step 2: Use the Template
Copy responsive template from `apps/admin/.instructions.md`

### Step 3: Apply Design System
- Colors: `#a9ff2e` (accent), `#d8ffe8` (text), `rgba(9,16,12,0.86)` (bg)
- Components: Use `Button`, `Card`, `Input` from `@/components/ui/`
- Dark theme: All backgrounds dark, all text light

### Step 4: Validate
```bash
npm run lint -w apps/admin
npm run build -w apps/admin
# Test on mobile: 320px
# Test on tablet: 768px
# Test on desktop: 1920px
```

## 📱 Responsive Breakpoints

```
Default (base)    → 320px+ (mobile)
sm:               → 640px+ (tablet portrait)
md:               → 768px+ (tablet landscape)
lg:               → 1024px+ (small desktop)
xl:               → 1280px+ (large desktop)
```

## 🎨 Design System Colors

```typescript
// Primary Colors
#a9ff2e    // Accent green (highlights, focus, hover)
#d8ffe8    // Primary text (light green)
#d4f8e2    // Secondary text (lighter green)
#dbff8b    // Positive indicator (fat %)

// Backgrounds
rgba(9,16,12,0.86)      // Card background (dark)
rgba(0,0,0,0.3)         // Expanded overlay (darker)
rgba(169,255,46,0.2)    // Border color (green)
rgba(169,255,46,0.05)   // Hover background (subtle)

// Text Colors
text-gray-400           // Secondary labels
text-gray-300           // Tertiary text
#ff7b7b                 // Error/negative (fat mass)

// Status Colors
#a9ff2e / green         // Success, positive
#ff7b7b / red           // Error, negative
#dbff8b / yellow        // Warning
text-gray-400 / gray    // Neutral, loading
```

## 📋 Mandatory Patterns

### Pattern 1: Responsive Grid
```typescript
<div className="grid grid-cols-1 sm:grid-cols-2 md:grid-cols-3 gap-3 sm:gap-4 md:gap-6">
```

### Pattern 2: Responsive Buttons
```typescript
<div className="flex flex-col sm:flex-row gap-2 sm:gap-3 w-full">
  <Button className="w-full sm:flex-1">Primary</Button>
  <Button variant="outline" className="w-full sm:flex-1">Secondary</Button>
</div>
```

### Pattern 3: Responsive Padding
```typescript
<div className="p-3 sm:p-4 md:p-6">
```

### Pattern 4: Responsive Typography
```typescript
<h1 className="text-lg sm:text-xl md:text-2xl font-bold text-[#d8ffe8]">
```

## ⚠️ Anti-Patterns (NEVER DO)

- ❌ Fixed widths: `w-[800px]`
- ❌ Only desktop grids: `grid-cols-4`
- ❌ No responsive spacing: `p-6 gap-4`
- ❌ Light text on light bg: `text-gray-200` on white
- ❌ Untyped props: `props: any`
- ❌ No accessibility: Missing `title`, `aria-*`
- ❌ Horizontal scrolling: Allowed anywhere

## 🎓 Example: From Request to Implementation

### User Request
```
"Create an evaluation metrics card"
```

### Process

1. **Plan Responsively**
   - Mobile: Single metric, stacked
   - Tablet: Side-by-side metrics
   - Desktop: 4-column grid

2. **Generate from Template**
   ```typescript
   <div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-4 gap-3 sm:gap-4">
     {metrics.map(metric => (
       <Card key={metric.id}>
         <CardContent className="p-3 sm:p-4">
           {/* Content */}
         </CardContent>
       </Card>
     ))}
   </div>
   ```

3. **Apply Design System**
   ```typescript
   <h3 className="text-sm sm:text-base font-semibold text-[#d8ffe8]">
   <p className="text-xs sm:text-sm text-gray-400">
   ```

4. **Validate**
   ```bash
   npm run lint -w apps/admin  # ✓
   npm run build -w apps/admin # ✓
   # Test on mobile/tablet/desktop
   ```

5. **Done!** ✅
